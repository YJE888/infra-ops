# 디지털플랫폼정부위원회 통합 테스트베드 구축 (NHN Cloud)

## 1. 프로젝트 개요
- 사업명: 디지털플랫폼정부위원회 통합 테스트베드 구축
- 수행 시기: 2024년 11월
- 클라우드 환경: NHN Cloud
- 주요 기술: Kubernetes
- 역할: 인프라 구축 및 쿠버네티스 기반 솔루션 설치·운영

본 프로젝트는 디지털플랫폼정부위원회 주관 통합 테스트베드 환경에 자사 SW 개발지원도구 솔루션을 설치·운영하기 위한 인프라 구축을 목표로 진행되었습니다.

---

## 2. 담당 역할
- NHN Cloud 환경 내 쿠버네티스 기반 인프라 구성
- Kubernetes 클러스터 환경에 자사 SW 개발지원도구 솔루션 설치
- 테스트베드 목적에 맞는 서비스 구동 및 운영 지원
- 초기 운영 안정성 확보를 위한 설정 검토 및 구성 보완

---

## 3. 수행 내용

### 3.1 쿠버네티스 환경 구성
- NHN Cloud에서 제공하는 Kubernetes 환경을 기반으로 서비스 운영 구조 검토
- 퍼블릭 Kubernetes 클러스터에 Ingress Controller(Nginx)를 구성하여 외부 트래픽을 수신하고, Ingress 규칙 기반으로 내부 서비스로 요청을 전달하는 구조로 운영
- 자사 솔루션이 정상 동작할 수 있도록 네임스페이스, 리소스 구성 확인
- 파드 배포 및 서비스 접근 구조 점검

### 3.2 자사 SW 개발지원도구 솔루션 설치
- Kubernetes 환경에 자사 솔루션 배포
- 컨테이너 이미지 및 실행 구성 확인
- 서비스 구동 상태 점검 및 기본 테스트 수행

### 3.3 테스트베드 운영 지원
- 테스트베드 환경 특성에 맞게 구성 값 조정
- 서비스 정상 동작 여부 모니터링

### 3.4 DNS 구성 제약 대응
- NHN Cloud 기본 네임서버에서 3-depth 도메인 레코드 등록이 지원되지 않는 제약 사항 확인
- 하위 도메인 단위로 NS 위임을 적용하여 3-depth 레코드 등록이 가능한 네임서버로 DNS 관리 권한을 분리
- 이를 통해 NHN 네임서버의 기능 제약을 우회하면서도 기존 도메인 구조는 유지

---

## 4. 작업 시 고려사항
- 공공 테스트베드 환경 특성상 안정성과 재현성을 우선 고려
- 향후 확장 및 추가 테스트를 고려한 기본 구성 유지
- 운영 환경과 유사한 형태로 쿠버네티스 기반 서비스 구성

## 5. 결과

- 테스트베드 목적에 맞는 서비스 구동 환경 확보
- 이후 테스트 및 검증을 진행할 수 있는 기반 인프라 구성 완료
